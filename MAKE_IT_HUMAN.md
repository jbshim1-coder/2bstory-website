# MAKE IT HUMAN — 투비스토리 AI 티 안 나는 사이트 만들기

> Claude Code는 코드·카피를 생성할 때마다 이 문서의 §1·§3 체크리스트를 점검한다.
> 발주처(대표님)가 "어, 이거 우리 회사 같다"고 느끼는 것이 목표다.

## 투비스토리의 정체성 — 한 줄 요약

```
한 단어:    검증 (proven)
세 단어:    광고대행사 중 책을 쓴 유일한 회사
다섯 형용사: 직설적 · 데이터 기반 · 현장감 있는 · 전문가적 · 담백한
금기:       자화자찬 · 감성팔이 · SaaS 영문 직역 · 모호한 형용사
```

**심진보 대표의 말투가 모든 카피의 기준이다.**
책 11권을 쓴 저자의 말투 — 과장 없이 현장 경험을 사실로 말하는 방식.

## §1. 카피·문구의 함정 (AI티 1순위)

### 🚫 투비스토리 사이트에서 절대 쓰지 말 것

```
[마케팅 대행사 AI 클리셰 — 가장 위험]
데이터 기반 마케팅     → "GA·GTM으로 매체 기여도까지 추적합니다"로
퍼포먼스 극대화        → 구체적 수치 또는 방법론으로
최적화                 → 뭘 어떻게 최적화하는지 명시
전략 수립              → 어떤 전략인지 구체적으로
통합 솔루션            → 절대 X
원스톱 서비스          → 절대 X
차별화된               → 빼버리기 (자화자찬)
업계 최고              → 검증 가능한 사실로 (저서 11권, 18년)
혁신적인               → 빼버리기
탁월한 성과            → 수치로 대체

[모호한 형용사]
다양한 경험            → 구체 숫자 (100여 개 기업, 18년)
풍부한 노하우          → "2008년부터"로
효율적인               → 구체적 효과로
고객 중심              → 빼버리기 (당연한 말)
맞춤형                 → 어떻게 맞추는지 설명
전문적인               → 증거로 보여주기 (저서, 케이스)

[SaaS·영문 직역티]
Empower your marketing
Data-driven insights
Maximize your ROI
Performance marketing solutions
→ 모두 한국어로 다시 씀

[감성팔이]
함께 성장하는          → 실제 케이스로
파트너십               → 구체적 협력 내용으로
소중한 고객            → 빼버리기
```

### ✅ 투비스토리 카피 5원칙

```
원칙 1. 18년과 숫자가 신뢰를 만든다
  ❌ "오랜 경험을 바탕으로"
  ✅ "2008년부터 100여 개 기업과 일했습니다"
  ✅ "18년간 매일 GA 데이터를 봤습니다"

원칙 2. 저서가 가장 강한 증거다
  ❌ "병원 마케팅 전문가입니다"
  ✅ "병원 마케팅 책을 5권 썼습니다"
  ✅ "책에서 말한 방법을 직접 운영합니다"

원칙 3. 광고주 의심에 먼저 답한다
  ❌ "최고의 결과를 약속드립니다"
  ✅ "광고비 100만원짜리도 GA로 매체 기여도를 추적합니다"
  ✅ "계약 기간에 묶이지 않습니다. 1개월 전 통보로 해지 가능합니다"

원칙 4. 강의와 운영이 같은 사람이라는 게 차별점이다
  ❌ "이론과 실무를 겸비한"
  ✅ "매일경제 강단에서 말한 방법론을 다음 날 캠페인에 씁니다"

원칙 5. 병원 이야기는 5권의 책으로 증명한다
  ❌ "병원 마케팅 경험이 풍부합니다"
  ✅ "비급여 마케팅 조언 100가지를 책으로 정리했습니다"
  ✅ "원장님이 직접 읽고 진단하는 자가진단 KIT도 출판했습니다"
```

## §2. 한국어 마이크로카피

### 폼·버튼·에러 문구 원칙

```
1. 버튼 텍스트는 행동 동사 + 명확한 결과
   ❌ "제출" "확인" "클릭"
   ✅ "문의하기" "백서 받기" "진단 신청하기"

2. placeholder는 예시
   label: "이메일"
   placeholder: "help@yourcompany.com"

3. 에러 메시지는 해결책 포함
   ❌ "필수 항목입니다"
   ✅ "이메일 주소를 입력해주세요 (예: name@company.com)"

4. 성공 메시지는 다음을 알려줌
   ❌ "완료되었습니다"
   ✅ "접수되었습니다. 1영업일 내에 담당 컨설턴트가 직접 연락드립니다"

5. 빈 상태(Empty State)도 투비스토리 톤으로
   ❌ "데이터가 없습니다"
   ✅ "아직 준비 중입니다. 곧 업데이트됩니다"
```

### 자주 틀리는 한국어 디테일

```
word-break: keep-all     ← 한글 단어 중간 끊김 방지 (필수)
line-height: 1.75        ← 한글 본문은 1.7~1.8 (영문 1.5 기준 금지)
letter-spacing: -0.02em  ← 한글 본문 살짝 조여야 자연스러움

마침표: 헤딩은 X, 본문 문장은 O
쉼표: 나열 시 마지막 전 쉼표 생략 (한국식)
숫자: "12명+" → "12명 이상" (공식 텍스트), "12+" (숫자 강조 시 OK)
```

## §3. 시각 안티 패턴 — Claude Code가 자동 추가하는 것들

### 🚫 투비스토리에서 차단할 패턴

| AI 자동 추가 | 차단 이유 | 투비스토리 대안 |
|---|---|---|
| 보라-파랑 그라디언트 배경 | 2024 SaaS 클리셰 | 단색 (#1F1F1F 또는 #F8F8F8) |
| 무지개 그라디언트 | 가장 명확한 AI티 | 절대 X |
| Glassmorphism | 투비 톤과 안 맞음 | flat 디자인 |
| 이모지 (🚀 ✨ 💡) | AI티 1순위 | Lucide React 아이콘만 |
| 카드 hover 3D 기울기 | gimmick | `hover:-translate-y-1`만 |
| 떠다니는 blob 도형 | 클리셰 | 정적 배경 |
| `rounded-3xl` 이상 모서리 | 만화책 느낌 | `rounded-md` `rounded-lg` |
| `shadow-2xl` 짙은 그림자 | 부유감 과다 | `shadow-sm` `shadow-md` |
| 모든 섹션 같은 `py-24` | AI 티 | 섹션마다 의도적 변화 |
| 모든 헤딩 가운데 정렬 | 단조로움 | 좌측 정렬 섞기 |
| Hero에 오른쪽 위 장식 도형 | 의미 없음 | 대표 영상 썸네일로 대체 |

### ✅ 투비스토리 허용 패턴

```
✅ 단색 히어로 배경 (#1F1F1F→#2D1208 그라디언트 — 이 범위만)
✅ 약한 카드 그림자 (shadow-sm / shadow-md)
✅ 카드 hover 위로 4px (`hover:-translate-y-1` 0.2s ease-out)
✅ 스크롤 진입 시 카운터 숫자 카운트업 (18 / 100 / 11 / 15)
✅ Fade-in + slide-up 섹션 등장 애니메이션 (subtle)
✅ 오렌지 (#D94A0F) 는 CTA 버튼·배지·강조에만 — 남발 금지
```

### 투비스토리만의 "예측 못한 디테일" 아이디어

사람이 만든 사이트처럼 보이게 하는 차별화 요소:
```
- 저서 11권 카드에 "투비스토리 출판" 오렌지 배지
- 숫자 카운터: 18년, 100+, 11권, 15명 — "+", "권", "명" 단위가 오렌지
- FAQ 아코디언 Q 앞 오렌지 번호 원형
- 케이스 스터디 카드 좌측 4px 오렌지 세로 바
- CTA 배너 3단계 버튼이 🟡 🟠 🔴로 명확히 구분
- 대표 영상 썸네일에 플레이 버튼 오버레이 (실제 대표가 나옴)
```

## §5. 변화의 의도 — 단조로움 회피

AI는 모든 섹션을 같은 구조로 만들려 한다. 8번 같은 패턴이 반복되면 즉시 AI티가 난다.

### HOME 7개 섹션의 의도적 리듬 변화

```
§01 히어로     — 풀스크린 다크 배경 + 비대칭 2열 (영상 좌 / 텍스트 우)
§02 4가지 약속 — 연한 오렌지 배경 + 4열 카드 (배경색 변화)
§03 숫자       — 흰 배경 + 숫자 4개 (카드 없음) ← 패턴 변화
§04 분야 카드  — 연회색 배경 + 2열 3행 카드
§05 광고주 로고— 흰 배경 + 탭 + 슬라이더 (인터랙션 변화)
§06 인사이트   — 연회색 배경 + 비대칭 (좌 칼럼 목록 / 우 영상) ← 변화
§07 CTA 배너   — 오렌지 배경 (섹션 전체 포인트 컬러) ← 강한 변화
```

### 산업 페이지는 공통 구조지만 강조점이 다름

```
병원: 도서 5권 슬라이더 + 원장님 책 출간 서비스 (타 페이지에 없음)
B2B:  12단계 번호 스텝 그리드 + 백서 다운로드 폼
가맹: "1호점 → 200호점" 진행 타임라인 강조
NGO:  후원 전환 3단계 깔때기 다이어그램
```

## 🎯 최종 자가 점검 체크리스트

Claude Code가 시안 생성 후, 또는 사람이 검수 전 점검:

### 카피 점검
- [ ] "다양한", "효율적인", "탁월한", "최적화" 등 모호한 형용사 없음
- [ ] "원스톱", "통합 솔루션", "차별화된" 없음
- [ ] 숫자로 대체 가능한 표현은 숫자로 교체됨
- [ ] 대표 저서·연혁·케이스가 근거로 쓰임

### 마이크로카피 점검
- [ ] 버튼이 행동 동사 + 결과 ("문의하기", "받기", "신청하기")
- [ ] 에러 메시지에 해결책 포함
- [ ] 성공 메시지에 다음 액션 안내

### 시각 점검
- [ ] 이모지 없음
- [ ] 그라디언트 배경 없음 (히어로 다크 제외)
- [ ] Glassmorphism·3D 효과 없음
- [ ] 오렌지는 CTA·배지·강조에만 (본문 텍스트에 남발 X)
- [ ] 모든 섹션이 같은 배경색·같은 레이아웃 아님

### 한국 디테일 점검
- [ ] 푸터에 사업자등록번호 (또는 [TODO] 명시)
- [ ] 연락처 한국식 표기 (070-8676-7132)
- [ ] 지도는 카카오맵
- [ ] 개인정보처리방침 Footer에 표시

### 타이포그래피 점검
- [ ] `word-break: keep-all` 적용
- [ ] 한글 본문 `line-height: 1.75` 이상
